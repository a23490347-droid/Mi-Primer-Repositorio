## ¿Qué es Git?

Git es una herramienta que sirve para llevar un control de versiones en un proyecto,
Básicamente ayuda a guardar cambios que vamos haciendo en los archivos y así poder ver
el historial de lo que se hizo,También permite regresar a versiones anteriores si algo
sale mal o si se borra algo por error.

## Por qué necesitamos Git

Git es necesario porque ayuda a que el trabajo sea más organizado y seguro.
Sin Git, sería fácil perder archivos o no saber cuál es la versión correcta del proyecto.
tambien, cuando se trabaja en equipo, Git ayuda a que todos puedan subir cambios sin que
se borre el trabajo de los demás.

## Los 3 conceptos clave de Git

1. **Repositorio:** Es la carpeta del proyecto donde Git guarda los archivos y también el historial de cambios.
2. **Commit:** Es como un guardado del proyecto, donde se registra lo que se modificó y se le pone un mensaje.
3. **Rama:** Es una copia del proyecto donde se pueden hacer cambios sin afectar la versión principal.

## Los comandos que aprendí

- `git init` - Sirve para iniciar Git dentro de una carpeta.
- `git add` - Sirve para seleccionar los archivos que quiero guardar en el commit.
- `git commit` - Guarda los cambios con un mensaje.
- `git push` - Sube los cambios a GitHub.
- `git pull` - Descarga cambios del repositorio de GitHub.
- `git status` - Muestra qué archivos han sido modificados o agregados.
- `git log` - Muestra el historial de commits realizados.

## Mi reflexión final

y organizada, y que no se pierde el trabajo fácilmente. También entendí que GitHub es muy
